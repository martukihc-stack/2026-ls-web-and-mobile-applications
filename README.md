# 📱 Mobile App & Web Development (Erasmus 2026)

Welcome to the Mobile App & Web Development course. This course is designed to take you through the full lifecycle of a digital product—from the first design wireframe to a live, tracked, and optimized application.

---

## 📋 Course Essentials
* **Duration:** 12 Sessions
* **Format:** 1.5h Practical Seminar followed by a 0.75h Lecture.
* **Language:** English
* **Requirement for Credit:**
    * **Attendance:** Minimum 80% (You can miss at most 2 session).
    * **Final Project:** Submission of the Final Report via GitHub Pull Request.
    * **Grading:** Pass/Fail (Credit based).

## 🎓 The Final Report

To receive credit, you must submit a short report documented in this repository.

### 🛠 Submission Requirements
1.  **Format:** A single Markdown (`.md`) or PDF file.
2.  **Folder:** Upload to the `/reports/` folder.
3.  **File Name:** `SURNAME_FIRSTNAME_REPORT.md` (or .pdf).
4.  **Submission Method:** **Pull Request (PR)**.
    * **PR Title Format:** `[ODD/EVEN] Report - Surname Firstname`
    * *Note: Specify if you are in the "Odd" or "Even" week class in the title.*

### 📝 Report Structure
The report should be concise. For **each** of the 6 themes, include:
* **Chapter Title:** (e.g., Week 4: Backend & Data).
* **Summary:** 2–3 sentences describing what you learned/built.
* **Evidence:** One screenshot (image) of your work (e.g., Figma design, Code snippet, or Live website).

## 🆘 Troubleshooting & Help
If something isn't working:
1.  **Check the DevTools:** Right-click > Inspect > Console (look for red errors).
2.  **Hard Refresh:** `Ctrl + Shift + R` (or `Cmd + Shift + R` on Mac).
3.  **Ask AI:** Use Claude or ChatGPT to explain the error code.
4.  **Raise an Issue:** Open a GitHub Issue in this repo if you find a bug in the course materials.
